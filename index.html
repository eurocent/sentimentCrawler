Sentimentcrawler

sentimentCrawler is a file or http crawler which will build subject, predicate, object triple representations for use in Sentiment Analysis tasks.

Welcome to sentimentCrawler

Sentiment Crawler is a software component generated around the Eurosentiment Platform. In particular Sentiment Analysis is described as a new discipline whose aim is the computational treatment of opinion, sentiment and subjectivity within texts.

The main concept of the Eurosentiment project is to provide a shared language resource pool for fostering sentiment analysis. Sentiment Crawler acts as one of many tools within the Eurosentiment Toolkit for achieving this goal.

SO... What is sentimentCrawler???

sentimentCrawler is an HTTP crawler which builds subject, predicate, object triple representations to automatically create corpora that can be used for sentiment analysis. This is done in a semi automatic fashion by leveraging vocuabularies aimed at achieving the Web of Data such as schema.org markup.

Examples of what kind of vocubularies and the topic areas sentimentCrawler utilises include:

  1. Events: An event happening at a certain time and location, such as a concert, lecture, or festival. Ticketing information may be added via the 'offers' property. Repeated events may be structured as separate Event objects.
  2. Organization: An organization such as a school, NGO, corporation, club, etc.
  3. Restaurant: Your favourite (or worst) place to eat.
  4. Product: Any offered product or service. For example: a pair of shoes; a concert ticket; the rental of a car; a haircut; or an episode of a TV show streamed online.
  5. Review: A review of an item - for example, of a restaurant, movie, electrical equipment, holiday or store.
  6. ... and many many more

The workflow for using sentimentCrawler is as follows:

  1. Fetch a web corpus by configuring the crawler (described below) - make sure you point it to a website with web markup such as reviews!
  2. Automatically get subject, predicate, object (RDF triples) relationships from implicit content and structure thanks to the standard (implicit) markup that should be in the pages.
  3. define an output representation/serialization format for persisting the generated triples
  4. write the triples to an output directory for use within Sentiment Analysis tasks

Current Features

The tool is extremely lightweight but there are some nice features:

  1. The output path for generated triples is configurable so that output can be stored anywhere locally on file system
  2. The output format is configurable and can be one of turtle, ntriples, rdfxml, nquads, trix or json
  3. The extraction of implicit structure is handled by Apache Any23, an actively maintained top level project at The Apache Software Foundation

How can I use sentimentCrawler?

Pretty easily... we did say it was lightweight after all didn't we!:

$ git clone https://github.com/eurocent/sentimentCrawler.git
$ cd crawler
$ mvn install
$ cd target
$ java -jar sentiment-crawler-0.0.1-SNAPSHOT-jar-with-dependencies.jar -url http://reviews.bestbuy.com/3545a/6141277/zagg-invisibleshield-for-samsung-galaxy-gear-2-neo-fit-watch-reviews/reviews.htm -outputDir /usr/local/sentimentCrawler/crawler/target -outputFormat turtle

In the example above, the output will be written to a the file located at /usr/local/sentimentCrawler/crawler/target/sentiment.txt
